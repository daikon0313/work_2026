## 📁 プロジェクト構造

このプロジェクトは以下のような構造になっています：

```
11_quiz_app/
├── src/               # ソースコード
│   ├── App.tsx       # メインコンポーネント
│   ├── main.tsx      # エントリーポイント
│   └── App.css       # スタイル
├── public/           # 静的ファイル
├── index.html        # HTML テンプレート
└── package.json      # 依存関係の管理
```

### 重要なファイル

**src/main.tsx**
- アプリケーションのエントリーポイント（最初に実行されるファイル）
- React アプリを HTML の要素にマウント（表示）する

**src/App.tsx**
- メインコンポーネント
- アプリケーションの画面を構成する

## 🚀 開発サーバーの起動

ターミナルで以下のコマンドを実行します：

```bash
npm run dev
```

ブラウザで `http://localhost:5173` を開くと、アプリケーションが表示されます。

### Hot Module Replacement (HMR)

ファイルを編集して保存すると、自動的にブラウザが更新されます。これを HMR と呼びます。

## 📝 React の基礎知識

### React とは？

React は、ユーザーインターフェース（UI）を構築するための JavaScript ライブラリです。

**特徴**
- コンポーネントベース：UI を部品（コンポーネント）に分割して開発
- 宣言的：「どう見えるべきか」を記述する（「どう変更するか」ではない）
- 学習しやすい：JavaScript の知識があれば始められる

### TypeScript とは？

TypeScript は、JavaScript に型を追加したプログラミング言語です。

**メリット**
- エラーを事前に発見できる
- コードの補完が効く
- チーム開発がしやすい

## 🔍 実際に見てみよう

### src/App.tsx を開く

```tsx
import { useState } from 'react'
import './App.css'

function App() {
  const [count, setCount] = useState(0)

  return (
    <>
      <h1>Vite + React</h1>
      <div className="card">
        <button onClick={() => setCount((count) => count + 1)}>
          count is {count}
        </button>
      </div>
    </>
  )
}

export default App
```

**このコードのポイント**
- `function App()`: これがコンポーネント（関数）
- `useState`: データの状態を管理するフック
- `return`: 画面に表示する内容（JSX）
- `onClick`: ボタンをクリックしたときの処理

## 📖 用語集

- **コンポーネント**: UI の部品。関数として定義される
- **JSX**: JavaScript の中に HTML のような記法を書ける構文
- **useState**: コンポーネントの状態を管理するフック
- **HMR**: ファイルを変更すると自動的にブラウザが更新される機能
