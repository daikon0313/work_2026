# 📅 週報システム

毎週の作業内容を自動で記録・整理するシステムです。

## 🚀 クイックスタート

### 週報の生成

```bash
cd /Users/d.harato/personal/2026/0_weekly_note
./shuuhou.sh
```

スクリプトを実行すると、今週の週報ファイルが自動生成されます。

## 📁 ファイル構成

```
0_weekly_note/
├── README.md          # このファイル
├── claude.md          # システム詳細説明
├── template.md        # 週報テンプレート
├── shuuhou.sh         # 週報生成スクリプト
└── 20260101.md        # 生成された週報（例）
```

## 📝 週報の命名規則

週報ファイル名: `YYYYMMWW.md`

- `YYYY`: 年（4桁）
- `MM`: 月（2桁）
- `WW`: その月の第何週目か（2桁）

**例**:
- `20260101.md` - 2026年1月第1週
- `20260102.md` - 2026年1月第2週
- `20260201.md` - 2026年2月第1週

## 📊 週報の内容

自動的に以下の情報が収集されます：

- ✅ **期間**: その週の開始日〜終了日（月曜〜日曜）
- 🎯 **Gitコミット**: 過去1週間のコミット一覧とサマリー
- 📋 **GitHub Issue**: クローズしたIssueと進行中のIssue

手動で追記する項目：

- 💬 **Claude Codeでのやり取り**: 主要な会話トピックと学び
- ✅ **次週のTODO**: 次週に取り組むタスクの整理

## ⚙️ 必要な環境

- **Git**: コミット履歴の取得
- **GitHub CLI** (`gh`): Issue情報の取得（オプション）
- **Bash**: スクリプトの実行

## 🔧 カスタマイズ

週報のフォーマットをカスタマイズするには `template.md` を編集してください。

## 💡 使い方のヒント

1. **定期実行**: 毎週土曜日にスクリプトを実行する習慣をつけましょう
2. **手動追記**: 生成後、Claude Codeでの重要なやり取りを追記
3. **次週の計画**: 次週のTODOを具体的に書き出すことで、週の始まりがスムーズに
