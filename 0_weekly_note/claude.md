# 週報システム

## 概要

このディレクトリは週次の作業内容を記録・整理するためのシステムです。

## ファイル命名規則

週報ファイルは以下の形式で命名されます：

- `20250101.md` - 2025年第1週（1月第1週）
- `20250102.md` - 2025年第2週（1月第2週）
- `20250201.md` - 2025年第5週（2月第1週）

フォーマット: `YYYYMMWW.md`
- `YYYY`: 年（4桁）
- `MM`: 月（2桁）
- `WW`: その月の第何週目か（2桁）

## 使用方法

### 週報の自動生成

毎週土曜日（または任意のタイミング）に以下のコマンドを実行：

```bash
cd /Users/d.harato/personal/2026/0_weekly_note
./shuuhou.sh
```

スクリプトは自動的に：
1. 今週の週番号を計算
2. 過去1週間のGitコミットを取得
3. GitHub Issueを取得
4. 週報ファイルを生成

### 週報の内容

各週報ファイルには以下の情報が含まれます：

- **期間**: その週の開始日と終了日
- **今週の成果**:
  - Gitコミットの一覧とサマリー
  - 主要な変更内容
- **GitHub Issue**:
  - クローズしたIssue
  - 進行中のIssue
- **Claude Code でのやり取り**:
  - 主要な会話トピック（手動で記入）
- **次週のTODO**:
  - 次週に取り組むタスクの整理

## ディレクトリ構造

```
0_weekly_note/
├── claude.md          # このファイル（システム説明）
├── shuuhou.sh         # 週報生成スクリプト
├── template.md        # 週報テンプレート
├── 20250101.md        # 2025年1月第1週の週報
├── 20250102.md        # 2025年1月第2週の週報
└── ...
```

## カスタマイズ

週報テンプレートは `template.md` を編集することでカスタマイズできます。

## 注意事項

- Claude Codeでのやり取りは自動取得が困難なため、週報生成後に手動で追記してください
- GitHub CLIがインストールされている必要があります（`gh` コマンド）
- Gitリポジトリのルートディレクトリから実行してください
