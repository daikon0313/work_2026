# 1_docs - ドキュメント管理

学習資料やドキュメントを管理するディレクトリです。

## プロジェクト概要

技術書、オンラインコース、記事などの学習資料を整理して管理します。

## ディレクトリ構成

```
1_docs/
├── books.md           # 読書リスト
├── ai_engineer/       # AI/MLエンジニア関連の資料
└── [その他カテゴリ]/  # テーマ別のドキュメント
```

## 使い方

### 読書リストの管理

`books.md` に読んだ本、読みたい本を記録します。

#### フォーマット例

```markdown
## 読了

- [ ] 書籍名 - 著者名
  - 読了日: YYYY-MM-DD
  - 評価: ★★★★☆
  - メモ: 重要なポイント

## 読書中

- [ ] 書籍名 - 著者名
  - 進捗: 50%
  - メモ: 気づき

## 読みたい

- [ ] 書籍名 - 著者名
  - 理由: なぜ読みたいか
```

### カテゴリ別ドキュメント

テーマごとにディレクトリを作成して、関連資料を整理します。

```
1_docs/
├── ai_engineer/        # AI/MLエンジニア関連
├── data_engineering/   # データエンジニアリング
├── cloud/              # クラウド技術
└── programming/        # プログラミング
```

## ベストプラクティス

### ドキュメント整理

1. **カテゴリ分け**
   - 技術分野ごとにディレクトリを作成
   - 関連性の高い資料をまとめる

2. **メモの取り方**
   - 重要なポイントを箇条書き
   - コードスニペットも含める
   - 参考URLを記録

3. **定期的な見直し**
   - 月に一度、読書リストを更新
   - 古い情報は整理

### Markdownファイルの命名規則

```
YYYY-MM-DD_タイトル.md         # 日付ベース
カテゴリ_トピック.md           # カテゴリベース
```

## 学習記録のテンプレート

```markdown
# タイトル

## 概要
資料の概要

## 学んだこと
- ポイント1
- ポイント2

## 実践メモ
実際に試したこと

## 参考リンク
- [リンク1](URL)
```

## 推奨ツール

- **Markdown エディタ**: VS Code, Obsidian
- **プレビュー**: GitHub, Markdown Preview
- **同期**: Git, Dropbox
